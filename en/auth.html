SSO Authentication Service

Implementation of an authentication service with support for OAuth 2.0 and OpenID Connect. At first on-premise IAM providers have been configured and connected. Then the service was integrated with own IaC CI/CD platform and with the authorizing router. Work was carried out on integration with own A2P message delivery service and a security audit service.

Implementation Details

The authentication service ensures registration of user device sessions (mobile, web, desktop), authentication, authorization, and management of user session lifecycles. The service is written in Golang and optimized for high performance.

There is a new functionality for user registration, issuance of JWT tokens (HMAC and RSA), authentication, and authorization has been implemented, including:

  • OAuth 2.0 and OpenID Connect
  • Certificate-based (X.509)
  • OTP-based
  • QR code-based
  • Username and password-based with MFA support.

The service supports integration with multiple identity providers (user account and role management), a message delivery platform, and an audit service (event logging, activity tracking, and security threat detection). Most services have a GraphQL administrative interface.

Client Authentication

Client-side authentication has been implemented and deployed for Android platforms using Kotlin (Rx), as well as for web and desktop platforms using React and Electron (RxJs).
